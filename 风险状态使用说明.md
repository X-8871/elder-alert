# 风险状态使用说明

## 适用范围

本说明基于 [风险判断与状态显示规则表_v1.md](C:/Users/22061/Desktop/Competition/github_sync_elder_alert/风险判断与状态显示规则表_v1.md) 整理，用来帮助老师、队友、演示人员或测试人员快速理解：

- 什么时候进入 `REMIND`
- 什么时候进入 `ALERT`
- 什么时候进入 `SOS`
- 什么时候恢复到 `NORMAL`

说明：

- 当前屏幕业务状态只有 `NORMAL / REMIND / ALERT / SOS`
- `REST` 只是内部“休息场景”上下文，不单独显示，也不单独上报
- 本文优先讲“真实场景版规则”，最后补充“实验版/演示版”的快速触发参数

## 一、四种业务状态是什么意思

### 1. `NORMAL`

表示当前没有触发需要提醒或报警的风险，系统处于正常监测状态。

### 2. `REMIND`

表示系统发现了“需要提醒老人关注”的情况，但还没有升级到强报警。

典型场景：

- 长时间无活动
- 持续高温
- MQ2 轻度烟雾/气体异常

### 3. `ALERT`

表示风险已经升级，或者已经出现更高危险程度，需要立即报警。

典型场景：

- `REMIND` 长时间无人处理
- MQ2 浓度达到高危险
- MQ2 异常同时伴随明显升温

### 4. `SOS`

表示老人主动求助。它不是系统推断出来的风险，而是老人主动按下 SOS 键触发的最高优先级状态。

## 二、什么时候进入 `REMIND`

### 情况 1：非休息场景下长时间无活动

进入条件：

- 当前不在休息场景
- 连续无活动超过 `5 分钟`

进入后含义：

- 系统认为老人可能长时间没有活动，需要先做提醒

恢复到 `NORMAL` 的方式：

- 检测到重新活动，并满足 `3 分钟内距离变化 >= 3 次`
- 或老人按 `OK`

如果一直不处理：

- `30 分钟` 无响应会从 `REMIND` 升级到 `ALERT`

### 情况 2：持续高温

进入条件：

- 温度 `>= 32°C`
- 持续 `5 分钟`

进入后含义：

- 系统认为环境温度持续偏高，需要提醒老人注意

恢复到 `NORMAL` 的方式：

- 温度回落后可恢复
- 或老人按 `OK` 先回到 `NORMAL`

补充规则：

- 按 `OK` 后会进入 `10 分钟` 冷却期
- 冷却期内不重复因为高温进入 `REMIND`
- 冷却结束后如果还是持续高温，可以再次进入 `REMIND`

### 情况 3：MQ2 轻度持续异常

进入条件：

- `mq2_filtered_raw >= 1200`
- 持续 `10 秒`

进入后含义：

- 系统怀疑存在烟雾或可燃气体异常，但危险程度还没有达到最高级

恢复到 `NORMAL` 的方式：

- `MQ2 < 1200`
- 并稳定一段时间，当前建议稳定 `5 分钟`

注意：

- 这种情况不建议只靠按 `OK` 就直接判定恢复正常

如果一直不处理：

- `30 分钟` 无响应，且浓度没有回落，会升级到 `ALERT`

## 三、什么时候进入 `ALERT`

### 情况 1：长时间无活动的提醒无人处理

进入条件：

- 先因“非休息场景长时间无活动”进入 `REMIND`
- 之后 `30 分钟` 无响应

恢复到 `NORMAL` 的方式：

- 老人按 `OK` 后可以直接恢复 `NORMAL`

### 情况 2：MQ2 轻度异常提醒后仍无人处理，且浓度未回落

进入条件：

- 先因 MQ2 轻度异常进入 `REMIND`
- 之后 `30 分钟` 无响应
- 且 MQ2 浓度仍未回落

恢复到 `NORMAL` 的方式：

- 允许按 `OK` 先解除到 `NORMAL`

补充规则：

- 按 `OK` 后进入 `5 分钟` 冷却期
- 冷却结束后如果浓度仍未回落，会再次进入 `ALERT`

### 情况 3：MQ2 高危险持续异常

进入条件：

- `mq2_filtered_raw >= 1500`
- 持续 `20 秒`

进入后含义：

- 系统认为烟雾/燃气异常已经达到高危险级别

恢复到 `NORMAL` 的方式：

- 不能只靠按 `OK` 直接判正常
- 应等待浓度回落并稳定后再解除

### 情况 4：MQ2 异常并伴随明显升温

进入条件：

- `mq2_filtered_raw >= 1200`
- 且 `1 分钟升温 >= 4°C`
- 或 `3 分钟升温 >= 5°C`

进入后含义：

- 系统认为这不仅是气体异常，还出现了火情发展迹象

恢复到 `NORMAL` 的方式：

- 不能只靠按 `OK` 直接判正常
- 应等待 MQ2 和温度同时回落并稳定后再解除

## 四、什么时候进入 `SOS`

进入条件：

- 老人主动按下 SOS 键
- 当前硬件入口是 `GPIO8`

进入后含义：

- 这是最高优先级求助状态
- 一旦触发，应立即显示 `SOS` 并立即上报

恢复到 `NORMAL` 的方式：

- 人工确认解除

## 五、什么时候恢复到 `NORMAL`

`NORMAL` 有两种来源：

- 系统刚启动时默认进入 `NORMAL`
- 风险解除后，从其他状态回到 `NORMAL`

常见恢复路径如下：

| 当前状态 | 恢复到 `NORMAL` 的条件 |
|---|---|
| `REMIND`（无活动） | 检测到重新活动，且 `3 分钟内距离变化 >= 3 次`；或老人按 `OK` |
| `REMIND`（高温） | 温度回落；或老人按 `OK` 先恢复，再进入高温冷却期 |
| `REMIND`（MQ2 轻度异常） | `MQ2 < 1200` 并稳定 `5 分钟` |
| `ALERT`（无活动升级） | 老人按 `OK` |
| `ALERT`（MQ2 轻度异常升级） | 允许按 `OK` 先恢复，但会进入冷却期；若浓度未回落，后面会再次报警 |
| `ALERT`（MQ2 高危险） | 需要等待浓度回落并稳定，不能只靠 `OK` |
| `ALERT`（MQ2 + 温升） | 需要等待 MQ2 与温度共同回落并稳定，不能只靠 `OK` |
| `SOS` | 人工确认解除 |

## 六、休息场景和状态的关系

休息场景不是屏幕上的一种状态，它只是为了减少误报。

进入休息场景的条件：

- `lux <= 20`
- 持续 `10 分钟`

进入休息场景后的作用：

- 不会因为“长时间无活动”进入 `REMIND`

退出休息场景的条件：

- `lux >= 50`
- 或检测到明显活动
- 并结合距离变化确认不是轻微抖动

当前建议判据：

- `3 分钟内距离变化 >= 3 次`
- 单次距离变化 `>= 20cm` 才计 1 次

补充：

- 夜间不开灯起夜时，也允许在低光下通过“持续明显活动”退出休息场景
- 当前临时工程值为：`1 分钟累计距离变化 >= 1.0m`

## 七、实验版/演示版下怎么更快看到状态变化

如果系统切到实验版/演示版参数，状态变化会明显加快，便于现场演示。

主要差异如下：

| 项目 | 真实场景版 | 实验版/演示版 |
|---|---:|---:|
| 无活动进入 `REMIND` | `> 5 分钟` | `> 30 秒` |
| 无活动 `REMIND` 升级 `ALERT` | `30 分钟` | `15 秒` |
| MQ2 轻度异常进入 `REMIND` | `>= 1200` 持续 `10 秒` | `>= 1000` 持续 `3 秒` |
| MQ2 轻度异常升级 `ALERT` | `30 分钟` | `15 秒` |
| MQ2 高危险进入 `ALERT` | `>= 1500` 持续 `20 秒` | `>= 1300` 持续 `5 秒` |
| 高温进入 `REMIND` | `>= 32°C` 持续 `5 分钟` | `>= 30°C` 持续 `30 秒` |
| 高温 `OK` 后冷却时间 | `10 分钟` | `1 分钟` |

使用提醒：

- 实验版只适合比赛演示、课堂演示和联调
- 不能把实验版阈值当成真实居家部署参数

## 八、最简理解版

可以把当前系统简单理解成下面这几条：

1. 默认状态是 `NORMAL`
2. 发现一般风险，先进入 `REMIND`
3. `REMIND` 长时间没人处理，或者风险本身已经很严重，就进入 `ALERT`
4. 老人主动求助时，直接进入 `SOS`
5. 风险消失、活动恢复、或人工确认后，系统再恢复到 `NORMAL`
