# 风险判断与状态显示规则表 v1

## 目的

本文件用于统一当前版本的：

- 风险判断规则
- TFT 状态显示规则
- 网络状态上报规则

本版本只记录当前已经确认的规则，不把未确认内容写成事实。

## 当前确认前提

- `REST` 不作为独立状态显示，也不作为独立网络上报状态。
- 休息场景只作为内部上下文使用，用来抑制“长时间无活动”的误报。
- 当前先采用以下参数：
  - 高温提醒阈值：`温度 >= 32°C`，持续 `5 分钟`
  - 进入休息场景：`lux <= 20`，持续 `10 分钟`
  - 退出休息场景：`lux >= 50`
  - 距离变化计数阈值：单次距离变化 `>= 20cm` 计为 `1 次`
  - 低光下明显活动退出休息的临时工程值：`1 分钟累计距离变化 >= 1.0m`
  - MQ2 轻度异常阈值：`T1 = 1200`
  - MQ2 高危险阈值：`T2 = 1500`
  - 高温提醒冷却时间：`10 分钟`

## 休息场景内部规则

休息场景不是独立业务状态，只作为风险判断上下文使用。

### 进入休息场景

- 条件：`lux <= 20`
- 持续时间：`10 分钟`
- 作用：进入休息上下文后，不因“长时间无活动”进入 `REMIND`

### 退出休息场景

以下两类条件都可以退出休息上下文，但优先级不同：

- 第一优先级：低光下明显活动退出
- 第二优先级：亮光 + 活动 + 距离变化退出

只要满足其中任一类退出条件，即可退出休息上下文，恢复正常无活动检测。

- 条件 1：`lux >= 50`
- 条件 2：检测到活动
- 条件 3：距离在短时间内多次变化
- 建议量化：`3 分钟内距离变化 >= 3 次`
- 单次距离变化 `>= 20cm` 计为 `1 次`

满足以上条件后，退出休息上下文，恢复正常无活动检测。

### 低光下的活动退出补充规则

考虑到老人可能在夜里不开灯起夜、摸黑活动，低光条件下也允许通过“持续明显活动”退出休息上下文。

- 条件 1：当前仍处于低光环境
- 条件 2：明确检测到有人活动
- 条件 3：距离发生较大改变，说明不是翻身、轻微抖动或单次误触发

当前建议量化为：

- 检测到人体存在
- 且 `3 分钟内距离变化 >= 3 次`
- 且单次距离变化 `>= 20cm` 计为 `1 次`
- 且累计距离变化幅度达到“明显变化”阈值

当前临时工程值定义为：

- `1 分钟累计距离变化 >= 1.0m`

该值当前作为临时工程值使用，后续仍建议结合毫米波距离数据实测进一步标定。

## 表 1：状态进入条件总表

| 目标状态 | 进入条件 | 数据/阈值 | 退出条件 | 用途 |
|---|---|---:|---|---|
| `NORMAL` | 系统启动默认状态；或风险解除后恢复正常 | 无 | 见表 2 | TFT 正常显示；正常周期上报 |
| `REMIND` | 非休息场景下连续无活动 | `> 5 分钟` | 1. 恢复活动且 `3 分钟内距离变化 >= 3 次` 自动回 `NORMAL` 2. 老人按 `OK` 回 `NORMAL` 3. `30 分钟` 无响应升级 `ALARM` | 风险提醒；TFT 提示确认；状态变化上报 |
| `REMIND` | 持续高温 | `温度 >= 32°C` 持续 `5 分钟` | 1. 老人按 `OK` 先回 `NORMAL` 2. 按 `OK` 后进入高温提醒冷却时间，当前建议 `10 分钟` 内不重复提醒 3. 冷却结束后若仍持续高温，可再次进入 `REMIND` | 环境风险提醒；TFT 提示确认；状态变化上报 |
| `REMIND` | MQ2 轻度持续异常 | `mq2_filtered_raw >= 1200` 持续 `10 秒` | `MQ2 < 1200` 并稳定一段时间后回 `NORMAL`，当前建议稳定 `5 分钟`；不能只靠 `OK` 直接判正常 | 疑似烟雾/气体异常提醒；状态变化上报 |
| `ALARM` | 无活动类 `REMIND` 后持续无人响应超时 | `30 分钟` | 老人按 `OK` 后可直接回 `NORMAL` | 升级报警；TFT 报警显示；立即上报 |
| `ALARM` | MQ2 轻度异常 `REMIND` 后持续无人响应且浓度未回落 | `30 分钟` | 允许按 `OK` 先解除到 `NORMAL`，并进入 `5 分钟` 冷却期；冷却结束后若浓度仍未回落，则再次进入 `ALARM` | 疑似风险持续存在并无人确认，升级报警；立即上报 |
| `ALARM` | MQ2 高危险持续异常 | `mq2_filtered_raw >= 1500` 持续 `20 秒` | 建议等待浓度回落并稳定后解除；不能只靠 `OK` 直接判正常 | 火灾/燃气异常主报警；立即上报 |
| `ALARM` | MQ2 异常并伴随明显升温 | 前提：`mq2_filtered_raw >= 1200`，且 `1 分钟升温 >= 4°C` 或 `3 分钟升温 >= 5°C` | 建议等待 MQ2 和温度共同回落并稳定后解除 | 火情发展增强证据；立即上报 |
| `SOS` | 老人主动按下 SOS 键 | `GPIO8` | 人工确认解除 | 最高优先级；TFT 显示 SOS；立即上报 |

## 表 2：从 NORMAL 进入 REMIND 的条件表

| 从 `NORMAL` 到 `REMIND` 的原因 | 进入条件 | 数据/阈值 | 自动恢复 `NORMAL` | `OK` 恢复 `NORMAL` | 是否上报 |
|---|---|---:|---|---|---|
| 非休息场景长时间无活动 | 当前不在休息上下文，且连续无活动 | `> 5 分钟` | 可以。检测到活动且 `3 分钟内距离变化 >= 3 次` | 可以 | 是 |
| 持续高温 | 温度持续偏高 | `温度 >= 32°C` 持续 `5 分钟` | 可以。温度回落后恢复；若已按 `OK` 回 `NORMAL`，则在建议 `10 分钟` 冷却时间结束后再重新评估是否再次进入 `REMIND` | 可以。按 `OK` 可先回 `NORMAL`，并进入建议 `10 分钟` 冷却时间 | 是 |
| MQ2 轻度持续异常 | 烟雾/气体达到轻度异常线 | `mq2_filtered_raw >= 1200` 持续 `10 秒` | 可以，但要求 `MQ2 < 1200` 并稳定；当前建议稳定 `5 分钟` | 不建议只靠 `OK` | 是 |

## 网络上报建议

当前建议上报以下状态变化：

- `NORMAL -> REMIND`
- `REMIND -> NORMAL`
- `REMIND -> ALARM`
- `任意状态 -> SOS`
- `SOS -> NORMAL`

休息场景不作为独立网络上报状态。

## 显示与判断关系

### 风险判断

本文件用于决定：

- 是否进入休息上下文
- 是否从 `NORMAL` 进入 `REMIND`
- 是否从 `REMIND` 升级到 `ALARM`
- 是否进入 `SOS`

### TFT 状态显示

当前业务显示状态仍为：

- `NORMAL`
- `REMIND`
- `ALARM`
- `SOS`

休息场景不单独显示为业务状态。

### 网络上报

网络上报以业务状态变化为主，不单独上报休息上下文切换。

## 当前仍保留为工程初值的项目

以下数据当前仅作为工程初值使用，后续仍建议通过实测进一步标定：

- `lux <= 20` / `lux >= 50`
- 单次距离变化 `>= 20cm`
- `3 分钟内距离变化 >= 3 次`
- `1 分钟累计距离变化 >= 1.0m`
- `MQ2 T1 = 1200`
- `MQ2 T2 = 1500`
- `1 分钟升温 >= 4°C`
- `3 分钟升温 >= 5°C`
- 高温提醒冷却时间 `10 分钟`
- MQ2 轻度异常升级后的 `ALARM` 冷却时间 `5 分钟`

这些值当前可用于状态机实现和联调，但不应表述为权威认证阈值。

## 实验版/演示版参数

以下参数仅用于比赛演示、课堂演示或联调验证，目的不是贴近真实老人居家场景，而是让状态变化更容易在短时间内被观察到。

使用要求：

- 演示时必须明确说明这是“实验版参数”
- 不能把以下数值写成真实部署参数
- 演示结束后应切回上文的真实场景规则

### 实验版参数表

| 项目 | 真实场景版 | 实验版/演示版 |
|---|---:|---:|
| 进入休息场景光照阈值 | `lux <= 20` 持续 `10 分钟` | `lux <= 30` 持续 `1 分钟` |
| 退出休息场景光照阈值 | `lux >= 50` | `lux >= 60` |
| 距离变化判定 | `3 分钟内 >= 3 次` | `30 秒内 >= 2 次` |
| 低光下明显活动退出休息 | 人体存在 + 距离明显变化 | 人体存在 + `30 秒内距离变化 >= 2 次`，且按工程值折算满足明显变化趋势 |
| 非休息场景无活动进入 `REMIND` | `> 5 分钟` | `> 30 秒` |
| 无活动类 `REMIND` 无响应升级 `ALARM` | `30 分钟` | `15 秒` |
| MQ2 轻度异常 `REMIND` 无响应升级 `ALARM` | `30 分钟` | `15 秒` |
| MQ2 轻度异常升级后 `OK` 冷却时间 | `5 分钟` | `30 秒` |
| 高温进入 `REMIND` | `>= 32°C` 持续 `5 分钟` | `>= 30°C` 持续 `30 秒` |
| 高温 `OK` 后冷却时间 | `10 分钟` | `1 分钟` |
| MQ2 轻度异常进入 `REMIND` | `>= 1200` 持续 `10 秒` | `>= 1000` 持续 `3 秒` |
| MQ2 高危险进入 `ALARM` | `>= 1500` 持续 `20 秒` | `>= 1300` 持续 `5 秒` |
| MQ2 + 温升联动进入 `ALARM` | `MQ2 >= 1200` 且 `1 分钟 +4°C` 或 `3 分钟 +5°C` | `MQ2 >= 1000` 且 `30 秒 +2°C` 或 `1 分钟 +3°C` |
| MQ2 轻度异常恢复 `NORMAL` | `< 1200` 稳定 `5 分钟` | `< 1000` 稳定 `10 秒` |

### 实验版用途说明

实验版参数适合演示以下流程：

- 人为遮光后较快进入休息上下文
- 开灯并活动后较快退出休息上下文
- 停止活动约半分钟后进入 `REMIND`
- 对无活动类或 MQ2 轻度异常类提醒，不处理约 `15 秒` 后升级 `ALARM`
- 通过短时升温或短时 MQ2 异常更快演示环境风险触发

### 实验版风险说明

实验版参数会显著提高误报概率，只适合：

- 现场答辩
- 课堂演示
- 代码联调
- 状态机验证

不适合：

- 独居老人真实居家长期运行
- 作为真实安全阈值对外宣传
